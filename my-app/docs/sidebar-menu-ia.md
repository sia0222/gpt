# 관리자 포털 사이드바 메뉴 구조안

## 1. 문서 목적

이 문서는 관리자 포털의 사이드바 메뉴 구조를 새 기준으로 정리한 문서이다.  
전체 메뉴는 사이드바 내 `2개의 큰 메뉴 그룹`으로 구성하며, 각 그룹 안에 세부 운영 메뉴를 배치한다.

## 2. 사이드바 기본 구조

- 사이드바에는 큰 메뉴 2개만 노출한다.
- 각 큰 메뉴는 펼침/접힘 구조로 설계할 수 있다.
- 사용자는 큰 메뉴를 선택한 뒤 하위 메뉴로 이동한다.
- 화면 설계 시 큰 메뉴 2개는 상단 고정 또는 아코디언 형태로 구성할 수 있다.

## 3. 큰 메뉴 구성

### 3.1 큰 메뉴 1: 운영 관리

운영과 내부 사용자 관리 중심의 메뉴 그룹이다.

하위 메뉴:
- 대시보드
- 대상자 관리
- 서비스 관리
- 위험 관리
- 문서 관리
- 회원 관리

설명:
- 대시보드: 전체 운영 현황과 우선 조치 현황 확인
- 대상자 관리: 대상자 목록, 상세, 기기 정보, 상담/조치 관리
- 서비스 관리: 대상자에게 제공되는 서비스 현황과 이력 관리
- 위험 관리: 이상 알림, 위험군, 조치 필요 대상 관리
- 문서 관리: 내부 운영 문서, 대상자 관련 문서, 첨부 자료 관리
- 회원 관리: 관리자 및 사용자 계정, 권한, 소속 관리

### 3.2 큰 메뉴 2: 포털 관리

대민포털 콘텐츠 및 데이터 운영 중심의 메뉴 그룹이다.

하위 메뉴:
- 게시판
- 정책
- 시설
- 통계
- 위기가구

설명:
- 게시판: 공지사항, 소식, FAQ, 자료실 등 관리
- 정책: 정책/약관/고정 콘텐츠 관리
- 시설: 지도 기반 시설 정보 및 카테고리 관리
- 통계: 통계 파일 업로드, 검수, 게시 관리
- 위기가구: 위기가구 관련 등록, 현황, 관리 데이터 운영

## 4. 사이드바 메뉴 예시

```text
[운영 관리]
 - 대시보드
 - 대상자 관리
 - 서비스 관리
 - 위험 관리
 - 문서 관리
 - 회원 관리

[포털 관리]
 - 게시판
 - 정책
 - 시설
 - 통계
 - 위기가구
```

## 5. 메뉴 설계 원칙

- `운영 관리`는 내부 운영 업무 중심으로 묶는다.
- `포털 관리`는 대민포털 노출 및 콘텐츠/데이터 관리 중심으로 묶는다.
- 사용자가 메뉴만 보아도 업무 성격을 바로 구분할 수 있어야 한다.
- 대상자 실무와 포털 운영이 섞여 보이지 않도록 그룹을 명확히 분리한다.
- 사이드바에서는 큰 메뉴 2개만 명확하게 보이고, 실제 화면 이동은 하위 메뉴를 통해 이루어지도록 한다.

## 6. 화면 적용 방향

- 기본 상태에서는 큰 메뉴 2개만 먼저 보이게 할 수 있다.
- 큰 메뉴를 클릭하면 하위 메뉴가 펼쳐지는 구조가 적합하다.
- 현재 선택된 큰 메뉴와 하위 메뉴는 활성 상태가 명확히 표시되어야 한다.
- 모바일에서는 큰 메뉴 기준 접힘 구조가 더 적합하다.

## 7. 한 줄 정리

- 관리자 포털 사이드바는 `운영 관리`와 `포털 관리`의 2개 큰 메뉴로 구성하고, 각각의 하위 메뉴를 통해 세부 기능으로 진입하는 구조로 설계한다.
