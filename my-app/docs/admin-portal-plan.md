# 관리포털 기획안

## 1. 프로젝트 개요

### 1.1 목적
- 대민포털과 연계되는 관리자 전용 운영 시스템을 구축한다.
- 대상자, 기기, 헬스케어 서비스, 콘텐츠, 통계 데이터를 통합 관리한다.
- 운영자가 실시간 상태를 빠르게 파악하고 이상 징후에 대응할 수 있도록 한다.
- 대민포털에 노출되는 게시판, 지도 정보, 카테고리, 통계 자료를 일관된 기준으로 관리한다.

### 1.2 기대효과
- 대상자 관리와 서비스 운영 프로세스 표준화
- 기기 상태 및 데이터 이상 징후의 조기 탐지
- 대민포털 콘텐츠 운영 효율 향상
- 운영 이력, 권한, 로그 기반의 책임 추적 가능

## 2. 사용자 및 권한 정의

### 2.1 사용자 유형
| 구분 | 설명 | 주요 권한 |
| --- | --- | --- |
| 시스템 관리자 | 전체 시스템 운영 책임자 | 전체 메뉴 접근, 사용자 관리, 권한 관리, 공통 설정 관리 |
| 운영 관리자 | 대상자 및 콘텐츠 운영 총괄 담당 | 대상자 관리, 통합 관제, 콘텐츠 관리, 통계 관리, 운영 정책 관리 |
| 동 담당자 | 담당 지역(동) 대상자 실무 담당 | 담당 대상자 조회/수정, 상담 및 조치 이력 등록, 서비스 상태 관리 |

### 2.2 권한 정책
- 메뉴 단위 접근 권한
- 조회, 등록, 수정, 삭제, 승인 권한 분리
- 개인정보 포함 화면은 별도 권한 그룹으로 분리
- 중요 작업은 이중 확인 및 로그 기록
- 담당 지역(동) 기준 데이터 접근 범위 제한 가능

### 2.3 사용자 유형별 권한 범위
| 메뉴/기능 | 시스템 관리자 | 운영 관리자 | 동 담당자 |
| --- | --- | --- | --- |
| 대시보드 | 전체 조회 | 전체 조회 | 담당 대상자 기준 조회 |
| 대상자 관리 | 전체 조회/수정 | 전체 조회/수정 | 담당 대상자 조회/수정 |
| 대상자 상세 내 기기/헬스케어 데이터 | 전체 조회/수정 | 전체 조회/수정 | 담당 대상자 조회/조치 등록 |
| 통합 관제 | 전체 조회/설정 | 전체 조회/처리 | 담당 대상자 관련 항목 조회 |
| 콘텐츠 관리 | 전체 관리 | 등록/수정/게시 | 조회 또는 제한적 참여 |
| 통계 관리 | 전체 관리 | 업로드/검수/게시 | 조회 또는 권한 제외 |
| 운영 관리 | 전체 관리 | 알림/이력 관리 | 본인 업무 관련 이력 조회 |
| 시스템 관리 | 전체 관리 | 제한 | 권한 없음 |

### 2.4 역할 적용 원칙
- 시스템 관리자는 시스템 설정과 권한을 관리하는 최상위 관리자이다.
- 운영 관리자는 대상자 운영과 콘텐츠/통계 운영을 총괄하는 실무 관리자이다.
- 동 담당자는 자신이 맡은 대상자를 중심으로 상태 확인, 상담 기록, 조치 이력 입력을 수행한다.
- 승인, 권한 변경, 공통 코드 변경 등 시스템 영향도가 큰 작업은 시스템 관리자 또는 운영 관리자 권한으로 제한한다.

## 3. 관리포털 핵심 목표

### 3.1 운영 관점 목표
- 대상자 현황을 한 화면에서 파악
- 실시간 기기 데이터 이상 여부 확인
- 헬스케어 서비스 제공 상태 및 이력 추적
- 대민포털 콘텐츠의 등록, 승인, 게시 상태 통제
- 업로드 기반 통계 자료의 정합성 관리
- 운영 영역별로 서로 다른 탐색 기준을 적용해 업무 효율을 높임

### 3.2 사용자 경험 목표
- 운영자가 3클릭 이내로 주요 대상자 상세에 도달
- 이상 상태 발생 시 우선순위별 알림 제공
- 상세 화면에서 조치 이력과 관련 데이터를 함께 조회
- 반복 운영 업무를 일괄처리 기능으로 최소화

## 4. 메뉴 구조(IA)

### 4.0 IA 설계 원칙
- 관리포털 전체를 하나의 기준으로 묶지 않고, 운영 영역별 탐색 단위를 다르게 설계한다.
- 대상자 운영 영역은 `대상자`를 기본 탐색 단위로 설정한다.
- 헬스케어 기기 4대의 상태와 측정값은 대상자 상세 화면 안에서 통합 조회한다.
- 콘텐츠 운영 영역은 `게시판, 배너, 지도, 통계 파일` 등 콘텐츠 자산을 기준으로 관리한다.
- 별도 관제 메뉴는 전체 대상자/전체 기기 이상 상황을 한 번에 보는 운영 보조 메뉴로 둔다.

### 4.1 1차 메뉴
1. 대시보드
2. 대상자 관리
3. 통합 관제
4. 콘텐츠 관리
5. 통계 관리
6. 운영 관리
7. 시스템 관리

### 4.2 IA 관점 구분
| 운영 영역 | 탐색 기준 | 설명 |
| --- | --- | --- |
| 대상자 운영 | 대상자 중심 | 대상자, 연결 기기 4대, 실시간 값, 헬스케어 이력, 상담/조치 이력을 함께 운영 |
| 통합 관제 | 상태/이상 중심 | 전체 대상자와 기기의 이상 상황을 한 번에 보고 대상자 상세로 이동 |

## 5. 핵심 기능 정의

## 5.1 대시보드

### 목적
- 운영 현황과 이상 징후를 빠르게 파악하는 상황판 역할

### 주요 위젯
- 전체 대상자 수
- 서비스 이용 중 대상자 수
- 정상/주의/위험 상태 대상자 수
- 연결 기기 수 및 오프라인 기기 수
- 최근 24시간 이상 알림 건수
- 미처리 알림 목록
- 우선 조치 대상자 목록
- 최근 업로드 통계 파일 현황
- 게시 예정/종료 예정 콘텐츠 현황

### 화면 특징
- 기간 필터, 지역 필터, 서비스 유형 필터 제공
- 사용자 유형별 조회 범위 차등 제공
- 긴급 알림 카드 상단 고정
- 대상자 상세 화면으로 즉시 이동 가능

## 5.2 대상자 관리

### 관리 항목
- 기본정보: 이름, 성별, 생년월일, 연락처, 주소, 보호자 정보
- 서비스정보: 이용 서비스, 담당 기관, 배정 담당자, 등록일, 상태
- 기기정보: 대상자별 연결 기기 4대, 설치일, 회수일, 상태
- 실시간 측정정보: 기기별 최근 수집값, 수집시각, 이상 여부
- 특이사항: 건강 유의사항, 메모, 주의 등급

### 주요 기능
- 대상자 검색 및 다중 조건 필터
- 개별 등록/수정/상태 변경
- 엑셀 일괄 등록 및 결과 검증
- 서비스 배정 및 해지
- 운영 관리자 또는 시스템 관리자의 배정 담당자 지정
- 좌측 목록에서 대상자 선택 시 우측 상세 화면 즉시 갱신
- 대상자별 기기 연결 관리
- 대상자 상세에서 기기 4대 상태 일괄 조회
- 대상자 상세에서 건강 측정값 추세 조회
- 상담 및 조치 이력 등록
- 동 담당자 업무 메모 및 후속 조치 입력

### 대상자 관리 화면 구성 원칙
- `대상자 관리` 메뉴는 단순 목록 화면이 아니라 `목록 + 상세가 결합된 작업공간`으로 설계한다.
- 좌측 영역은 검색, 필터, 대상자 목록을 배치한다.
- 우측 영역은 선택된 대상자의 상세 정보를 탭 구조로 배치한다.
- 대시보드의 우선 조치 대상자나 통합 관제에서 진입할 경우 동일한 작업공간에서 해당 대상자가 선택된 상태로 열리도록 한다.

### 대상자 상세 화면 핵심 구성
- 요약 카드: 대상자 상태, 서비스 상태, 최근 알림, 주의 등급, 배정 담당자
- 연결 기기 현황 탭: 기기 4대별 연결 상태, 배터리, 마지막 수신 시각
- 실시간 값 탭: 기기별 최근 측정값과 정상/주의/위험 판정
- 헬스케어 이력 탭: 기간별 측정 추이, 이상치 발생 이력
- 서비스/상담 탭: 제공 중 서비스, 상담 기록, 후속 조치 상태
- 메모/첨부 탭: 운영 메모, 파일, 특이사항 기록

### 상태 예시
- 신청
- 승인대기
- 이용중
- 일시중지
- 종료

## 5.3 통합 관제

### 관리 대상
- 웨어러블 기기
- 환경 센서
- 응급 호출 기기
- 헬스케어 측정 기기

### 모니터링 항목
- 대상자별 위험 상태
- 마지막 수신 시각
- 온라인/오프라인 상태
- 배터리 잔량
- 통신 상태
- 센서 측정값
- 임계치 초과 여부

### 주요 기능
- 이상 대상자 우선순위 조회
- 기기 목록 조회 및 상태 필터
- 실시간 데이터 스트림 확인
- 이상 상태 탐지 규칙 적용
- 장애/점검 이력 관리
- 대상자와 기기 매핑 정보 관리

### 운영 목적
- 전체 대상자를 한눈에 관제하되, 이상 발생 시 최종 도착점은 항상 대상자 상세 화면이 되도록 한다.
- 즉, 통합 관제는 `전체 상황판`, 대상자 관리는 `개별 운영 화면` 역할을 맡는다.

### 이상 탐지 예시
- 일정 시간 이상 데이터 미수신
- 배터리 임계치 이하
- 측정값 이상치 발생
- 장치 등록정보 불일치

## 5.4 대상자 기반 헬스케어 관리

### 관리 대상
- 대상자별 건강 측정 데이터
- 서비스 제공 이력
- 상담 기록
- 후속 조치 내역

### 주요 기능
- 대상자 상세 내 건강 데이터 시계열 조회
- 기간별 평균/최대/최소 값 확인
- 임계치 초과 이력 조회
- 상담 등록 및 배정 담당자 지정
- 조치 상태 관리(접수, 진행중, 완료)
- 서비스 이용 현황 및 이탈 현황 파악

### 조회 예시
- 혈압, 혈당, 심박수, 체온, 활동량, 수면 데이터
- 최근 7일/30일 트렌드
- 이상 징후 발생 후 조치 완료까지 소요 시간

### 설계 원칙
- 건강 데이터는 별도 독립 객체로 관리하되, 운영자 UI에서는 반드시 대상자 맥락 안에서 보이도록 한다.
- 헬스케어 기기 4대에서 들어오는 값은 기기별 구분이 가능해야 하지만, 최종 조회 기준은 대상자 중심으로 통합한다.

## 5.5 콘텐츠 관리

### 관리 범위
- 게시판(공지사항, 소식, FAQ, 자료실)
- 대민포털 메인 배너 및 팝업
- 지도 카테고리 및 지도 데이터
- 약관/정책 등 고정 페이지

### 게시판 관리 기능
- 게시글 등록, 수정, 삭제
- 임시저장, 예약게시, 게시종료
- 첨부파일 관리
- 상단 고정 및 노출 순서 지정
- 카테고리 및 태그 설정

### 지도 카테고리 관리 기능
- 카테고리 등록 및 순서 변경
- 카테고리별 마커/아이콘 설정
- 지도 데이터 활성/비활성 처리
- 대민포털 노출 여부 설정

### 운영 편의 기능
- 미리보기 제공
- 게시 이력 및 복원
- 승인 프로세스 적용 가능
- 운영 관리자 중심 게시 승인 흐름 적용 가능

### 설계 원칙
- 콘텐츠 운영은 대상자 중심이 아니라 콘텐츠 자산 중심으로 관리한다.
- 운영자는 게시물, 배너, 지도 카테고리, 노출 정책 자체를 직접 관리할 수 있어야 한다.
- 대상자 운영 데이터와 콘텐츠 운영 데이터는 같은 포털 안에 있더라도 탐색 구조를 분리하는 것이 적절하다.

## 13. 한 줄 정리
- 본 관리포털은 `대상자 운영은 대상자 중심`, `콘텐츠/통계 운영은 자산 중심`으로 분리해 관리하는 것이 핵심이다.
- 사용자 유형은 `시스템 관리자`, `운영 관리자`, `동 담당자`의 3개 축으로 단순화해 권한과 업무 범위를 명확히 구분한다.
