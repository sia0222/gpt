# 관리자 포털 페이지 구성 가이드

## 1. 문서 목적

이 문서는 관리자 포털의 전체적인 페이지 구성과 화면 간 흐름을 정리하기 위한 문서이다.  
디자인 스타일, 색상, 컴포넌트 형태, 스크립트 구조, 파일 네이밍은 고정하지 않으며 자유롭게 설계할 수 있다.  
다만 운영 흐름상 반드시 필요한 화면 구성 요소와 정보 구조는 유지하는 것을 원칙으로 한다.

## 2. 구성 원칙

- 대상자 운영 영역은 `대상자 중심`으로 구성한다.
- 콘텐츠 및 통계 운영 영역은 `콘텐츠/자산 중심`으로 구성한다.
- 대시보드는 전체 현황과 우선 조치 대상을 빠르게 파악하는 화면이어야 한다.
- 대상자 관리는 목록과 상세가 분리되지 않고 하나의 작업공간으로 이어지는 구조가 적합하다.
- 통합 관제는 전체 이상 상태를 파악하는 보조 운영 화면이며, 필요 시 대상자 작업공간으로 이동할 수 있어야 한다.
- 디자인은 자유롭게 구성하되, 운영자가 빠르게 판단하고 이동할 수 있는 정보 우선순위는 유지해야 한다.

## 3. 주요 페이지 구성

### 3.1 대시보드

대시보드는 전체 운영 상태를 한 화면에서 보는 메인 진입 페이지이다.

필수 구성 요소:
- 상단 현재 위치 및 사용자 정보 영역
- 공통 검색/필터 영역
- KPI 또는 핵심 현황 요약 영역
- 우선 조치 대상자 영역
- 실시간 관제 또는 상태 요약 영역
- 알림/장애 현황 영역
- 대상자 운영 현황 영역
- 콘텐츠 운영 현황 영역
- 통계 운영 현황 영역
- 최근 작업 이력 또는 최근 변경사항 영역

역할:
- 운영자가 접속 후 가장 먼저 보는 메인 현황판
- 위험 대상자와 미처리 알림을 빠르게 확인하는 시작점
- 대상자 관리, 통합 관제, 콘텐츠 관리 등으로 이동하는 허브 역할

### 3.2 대상자 관리 작업공간

대상자 관리는 단순 목록 페이지가 아니라 `목록 + 상세가 결합된 작업공간`으로 구성한다.

필수 구성 요소:
- 대상자 검색 영역
- 상태/지역/서비스/배정 담당자 기준 필터 영역
- 대상자 상태 요약 영역
- 대상자 목록 영역
- 선택된 대상자의 상세 요약 영역
- 상세 정보 탭 또는 섹션 영역

상세 정보에 포함되어야 하는 항목:
- 기본정보
- 보호자 및 운영 정보
- 연결 기기 4대 현황
- 실시간 수집값
- 헬스케어 이력
- 서비스 이용 현황
- 상담 및 조치 이력
- 운영 메모

역할:
- 운영 관리자와 동 담당자가 가장 많이 사용하는 실무 화면
- 대상자를 선택하고 즉시 상세 정보와 조치를 함께 수행하는 화면
- 대시보드나 통합 관제에서 특정 대상자 중심으로 이어지는 최종 작업 화면

### 3.3 통합 관제

통합 관제는 전체 대상자와 기기의 이상 상태를 모아서 보는 화면이다.

필수 구성 요소:
- 이상 대상자 목록
- 기기 상태 요약
- 미수신/오프라인 현황
- 이상치 발생 현황
- 장애/점검 이력
- 대상자 작업공간 이동 진입점

역할:
- 전체 상황을 한 번에 보고 우선 대응 대상을 가려내는 화면
- 대상자 개별 조치가 필요한 경우 대상자 관리 작업공간으로 연결

### 3.4 콘텐츠 관리

콘텐츠 관리는 대민포털에 노출되는 자산을 운영하는 화면이다.

필수 구성 요소:
- 게시판 관리
- 배너/팝업 관리
- 지도 카테고리 관리
- 노출 상태 및 게시 일정 관리
- 변경 이력 또는 승인 상태 확인 영역

역할:
- 대상자 운영과 별개로 콘텐츠 자산 자체를 관리하는 화면

### 3.5 통계 관리

통계 관리는 파일 업로드, 검수, 게시 상태를 관리하는 화면이다.

필수 구성 요소:
- 파일 목록
- 업로드 영역
- 검수 상태 영역
- 버전 관리 영역
- 게시 상태 및 노출 기간 관리 영역

역할:
- 통계 파일의 등록부터 게시까지 운영 흐름을 담당하는 화면

## 4. 페이지 간 흐름

기본 운영 흐름은 아래와 같다.

1. 대시보드에서 전체 현황 확인
2. 우선 조치 대상자 또는 메뉴를 통해 대상자 관리 진입
3. 대상자 관리 작업공간에서 대상자 선택
4. 대상자 상세 정보 확인 및 상담/조치 수행
5. 필요 시 통합 관제에서 전체 이상 상태 재확인
6. 별도 업무로 콘텐츠 관리 또는 통계 관리 수행

## 5. 사용자 유형별 관점

### 5.1 시스템 관리자
- 전체 메뉴 접근
- 시스템 설정, 계정, 권한, 공통 관리 기능 중심
- 전체 데이터 조회 가능

### 5.2 운영 관리자
- 대상자 운영, 통합 관제, 콘텐츠 운영, 통계 운영 중심
- 전체 대상자와 전체 운영 현황을 기준으로 업무 수행

### 5.3 동 담당자
- 자신이 맡은 대상자 중심으로 업무 수행
- 대상자 상태 확인, 상담 등록, 조치 이력 관리 중심
- 콘텐츠/통계 영역은 축소 조회 또는 제외 가능

## 6. 디자인 자유도 원칙

- 레이아웃은 좌우형, 카드형, 탭형, 대시보드형 등 자유롭게 구성할 수 있다.
- 색상, 타이포그래피, 인터랙션 스타일도 자유롭게 설계할 수 있다.
- 컴포넌트 명칭, 클래스 명, 스크립트 구조, 파일 구조 역시 고정하지 않는다.
- 단, 아래 요소는 유지해야 한다.

유지해야 하는 요소:
- 대시보드가 메인 현황판 역할을 해야 함
- 대상자 관리가 목록과 상세를 함께 다루는 작업공간이어야 함
- 대상자 상세에 기기 4대와 실시간 값, 서비스/상담 이력이 포함되어야 함
- 통합 관제가 전체 이상 상황을 모아서 볼 수 있어야 함
- 콘텐츠/통계 관리가 대상자 운영과 분리된 운영 축이어야 함

## 7. 한 줄 정리

- 이 문서는 화면의 모양을 고정하는 문서가 아니라, 관리자 포털이 어떤 페이지 구조와 운영 흐름을 가져야 하는지를 정리한 페이지 구성 가이드이다.
