# 관리포털 대시보드 화면구성안

## 1. 화면 개요

- 화면명: 관리포털 통합 대시보드
- 목적: 대상자 운영 현황, 이상 알림, 기기 상태, 콘텐츠/통계 운영 현황을 한 화면에서 확인
- 기본 기준: `대상자 운영은 대상자 중심`, `콘텐츠/통계 운영은 자산 중심`
- 기본 해상도 기준: Desktop 1440px 이상

## 2. 화면 구성 원칙

- 상단에서는 전체 상황을 빠르게 파악할 수 있어야 한다.
- 중단에서는 즉시 조치가 필요한 대상자와 실시간 상태를 우선 노출한다.
- 하단에서는 운영자가 후속 업무를 처리할 수 있도록 콘텐츠/통계/작업 이력을 보여준다.
- 동 담당자는 자신의 담당 지역(동) 기준으로만 조회되도록 필터링 가능해야 한다.

## 4. 영역별 상세 구성

### 4.1 상단 헤더
- 현재 위치: `관리포털 > 대시보드`
- 최종 갱신 시각 표시
- 사용자명과 사용자 유형 표시
- 빠른 이동 버튼 선택 가능
- 예: `대상자 관리`, `통합 관제`, `콘텐츠 관리`

### 4.2 공통 필터 바
- 기간 필터
- 지역 필터
- 담당 지역(동) 필터
- 서비스 유형 필터
- 대상자 상태 필터
- 사용자 유형에 따라 필터 노출 범위를 다르게 적용

### 4.3 KPI 카드 영역
- 전체 대상자 수
- 서비스 이용 중 대상자 수
- 위험 대상자 수
- 오프라인 기기 수
- 미처리 알림 수
- 오늘 게시 예정 콘텐츠 수

### 4.4 우선 조치 대상자 영역
- 위험도 순으로 정렬
- 대상자명, 이상 유형, 발생 시각, 배정 담당자 표시
- 클릭 시 대상자 관리 작업공간 내 해당 대상자 상세로 이동
- 동 담당자는 본인 담당 대상자만 노출

### 4.5 실시간 관제 현황 영역
- 대상자 상태 분포
- 기기 상태 분포
- 최근 수집 데이터 추이
- 이상 상태 발생 비율
- 통합 관제로 이동하는 진입점 역할

### 4.6 알림/장애 패널
- 긴급 알림
- 주의 알림
- 최근 장애 이력
- 미처리 알림 우선 노출
- 알림 클릭 시 대상자 관리 작업공간 또는 통합 관제 상세로 이동

### 4.7 대상자 운영 현황 영역
- 상태별 대상자 수
- 서비스 이용 현황
- 지역별 대상자 분포
- 배정 담당자별 관리 현황

### 4.8 헬스케어 데이터 현황 영역
- 혈압, 심박, 체온 등 주요 측정 항목 이상치 건수
- 최근 7일 주요 지표 추이
- 조치 완료 평균 시간
- 이상치 발생 후 미조치 건수

### 4.9 기기 운영 현황 영역
- 기기 유형별 연결 수
- 오프라인 기기 수
- 배터리 부족 기기 수
- 점검 중 기기 현황

### 4.10 콘텐츠 운영 현황 영역
- 게시판 게시 예정/종료 예정 건수
- 배너/팝업 노출 현황
- 지도 카테고리 변경 현황
- 콘텐츠 승인 대기 건수

### 4.11 통계 운영 현황 영역
- 최근 업로드 파일
- 검수 대기 파일
- 게시 중 통계 버전
- 반려 파일 현황

### 4.12 최근 작업 이력 영역
- 대상자 수정 이력
- 상담/조치 등록 이력
- 콘텐츠 게시 이력
- 통계 업로드/검수 이력

## 5. 사용자 유형별 대시보드 차이

### 5.1 시스템 관리자
- 전체 데이터 조회
- 권한 변경, 시스템 설정, 공통 코드 관련 위젯 확장 가능
- 전체 작업 이력과 감사 로그 접근

### 5.2 운영 관리자
- 대상자 운영과 콘텐츠/통계 운영 위젯 전체 조회
- 우선 조치 대상자, 알림 처리, 게시 승인 업무 중심
- 실무 운영 화면으로 가장 적합한 기본 대시보드

### 5.3 동 담당자
- 담당 지역(동) 대상자 기준으로 자동 필터 적용
- 우선 조치 대상자, 상담/조치 대상, 담당 대상자 상태 변화 중심으로 노출
- 콘텐츠/통계 영역은 조회 축소 또는 숨김 가능

## 6. 권장 컴포넌트 배치

### 6.1 1행
- 필터 바
- KPI 카드 6개

### 6.2 2행
- 좌측: 우선 조치 대상자 리스트
- 중앙: 실시간 관제 차트
- 우측: 알림/장애 패널

### 6.3 3행
- 대상자 운영 현황
- 헬스케어 데이터 현황
- 기기 운영 현황

### 6.4 4행
- 콘텐츠 운영 현황
- 통계 운영 현황
- 최근 작업 이력

## 7. 인터랙션 제안

- KPI 카드 클릭 시 관련 목록 화면으로 이동
- 좌측 메뉴 `대상자 관리` 클릭 시 대상자 관리 작업공간으로 이동
- 우선 조치 대상자 클릭 시 대상자 관리 작업공간에서 해당 대상자 선택 상태로 진입
- 알림 클릭 시 처리 상태 변경 가능
- 차트 클릭 시 필터가 적용된 상세 목록으로 이동
- 대시보드는 1분 또는 5분 주기 자동 새로고침 선택 가능

## 8. 화면 설계 메모

- `대상자 중심 운영`과 `콘텐츠/통계 운영`이 한 화면에 같이 있어도 섹션은 명확히 분리해야 한다.
- 가장 중요한 영역은 `우선 조치 대상자`, `실시간 관제`, `알림 패널`이다.
- 콘텐츠/통계 영역은 하단 배치가 적절하다.
- 동 담당자용 화면에서는 하단 콘텐츠/통계 영역을 축소한 전용 대시보드 버전도 고려할 수 있다.

## 9. 후속 작업 제안

- 대시보드 와이어프레임 시안
- 대상자 관리 작업공간(목록 + 상세) 시안
- 컴포넌트별 기능정의서
- 사용자 유형별 화면 권한정의서
- 통합 관제 화면 시안
